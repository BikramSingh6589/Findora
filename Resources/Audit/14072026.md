### Features Added
- Added QR Code scanning and generation specifically tailored to peer-to-peer resolutions and admin mediations in the chat system.
- Re-enabled QR system for claimant so that they can see a QR code once the item handover is mutually confirmed.
- Added a scanner button for the finder (even on mobile views) so they can successfully scan the claimant's QR code.

### Improvements
- Refined the chat logic so the screen no longer completely hides the QR system when the chat is frozen due to a resolved status. 
- Distinguishing chat statuses clearly. When a claim is fully resolved, the finder is instructed to scan the claimant's QR code, and the claimant is instructed to display it.
- Improved how admin mediation handles resolutions. When the admin decides to mitigate, the QR code generation is omitted entirely to follow the specific admin workflow. Both the claimant and finder now see an updated status indicating the issue has been mediated to the admin.

### Bug Fixes
- Fixed the issue where the QR system disappeared completely after the finder clicked the resolve button.
- Resolved an issue in the backend where admin resolutions accidentally created unused QR codes.

### User Experience
- The claimant and finder can securely hand over items peer-to-peer again. The founder will simply press the "Scan Claimant QR" button on mobile or desktop to finish the handover, while the claimant easily displays the code from their chat window.
- If admin mediates, users will immediately see an AI-protected handover chat indication stating the admin will resolve the case, without confusing them with irrelevant QR codes.

### Testing
- As a claimant, verify that after the finder confirms ownership, you can see a "Secure Handover QR Code" in your chat panel.
- As a finder, verify that when the claim is mutually resolved, you can see a "Scan Claimant QR" button within the chat area.
- For admin mediation, request mediation from the chat header. Wait for admin approval, and ensure neither the QR code nor the scanner appears, but instead, a message indicating "Mediated to Admin" appears.

### Notes
- Kept the "Chat is closed" input bar intact to make sure users understand that further chat messages are blocked on resolved/mediated claims.

### Time

### Features Added
- Implemented "Login Session Save" (Remember Me) functionality across User and Admin login flows. 
- Introduced a "Remember this device for 30 days" checkbox on the main login screen.

### Improvements
- Adjusted token storage logic in `AuthContext` to support both `sessionStorage` and `localStorage`.
- By default, user sessions now securely expire when the browser tab/window is closed (`sessionStorage`).
- If "Remember Me" is selected during login, the session persists across browser restarts (`localStorage`).
- Ensures OTP Verification gracefully inherits the "Remember Me" preference selected on the initial login screen.

### Bug Fixes
- None for this feature.

### User Experience
- Enhances security for users on public or shared campus computers, as simply closing the browser will log them out unless they explicitly opt to be remembered.
- Provides convenience for users on personal devices who want to stay logged in without repeated verifications.

### Testing
- Log in to the application *without* checking "Remember Me". Close the browser tab and reopen it. You should be logged out.
- Log in to the application and *check* "Remember Me". Close the browser tab and reopen it. You should remain logged in.
- Try the same flow for the Admin Portal login.
- Test logging out to ensure tokens are purged from both session and local storage.

### Notes
- Session storage enhances security significantly against session hijacking on shared university computers.
